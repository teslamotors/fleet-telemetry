# Docker Compose PostgreSQL - Build Setup Guide

## ✅ What Changed

The `docker-compose.postgres.yml` now **automatically builds** the Flask Telemetry app from your source code using the Dockerfile. You no longer need to pre-build the image.

---

## 🚀 Quick Start (Recommended)

### Prerequisites
- Docker and Docker Compose installed
- TLS certificates generated: `go run tools/main.go`

### One-Command Setup

```bash
cd /home/user/share/fleet-telemetry

# Generate TLS certificates (one-time)
go run tools/main.go

# Start everything (builds app + starts PostgreSQL)
docker-compose -f docker-compose.postgres.yml up -d

# Verify it's running
docker-compose -f docker-compose.postgres.yml ps
```

**That's it!** The app builds and starts automatically.

---

## 📋 How It Works

### Docker Compose Build Process

```yaml
app:
  build:
    context: .              # Uses current directory (fleet-telemetry)
    dockerfile: Dockerfile  # Uses the Dockerfile in project root
```

### Dockerfile Flow

1. Builds `libsodium` (dependency for ZMQ)
2. Builds `libzmq` (ZMQ library)
3. Copies project files
4. Runs `make` (which builds the binary via cmd/main.go)
5. Creates distroless image with the compiled binary

**Result:** `/go/bin/fleet-telemetry` binary ready to run

## ✨ What Gets Built

### Stage 1: Build Container
- Compiles `libsodium` from source
- Compiles `libzmq` from source  
- Copies your fleet-telemetry source code
- Runs `make` to build the binary
- Creates `/go/bin/fleet-telemetry`

### Stage 2: Runtime Container
- Minimal distroless image (very small)
- Only contains: Fleet Telemetry binary + runtime libs
- No build tools, no source code

This creates a ~50MB production-ready image!

## 🔄 Development Workflow

### After Code Changes

1. **Update code** (e.g., modify postgres.go)

2. **Rebuild and restart**
```bash
docker-compose -f docker-compose.postgres.yml up -d --build
```

3. **View new logs**
```bash
docker-compose -f docker-compose.postgres.yml logs -f app
```

### Building vs Pre-Built Image

| Aspect | Pre-Built | Build (Current) |
|--------|-----------|-----------------|
| **Initial setup** | `docker pull ...` | `docker build ...` |
| **After code change** | Must rebuild manually | Auto-builds with `--build` |
| **Size** | ~100MB per version | ~50MB final |
| **Storage** | Need image registry | Built locally |
| **Dependencies** | Pre-compiled | Compiled on build |

## 📁 What the Dockerfile Does

```dockerfile
# Stage 1: Build
FROM golang:1.23-bullseye AS build

# Download and compile libsodium
RUN wget https://github.com/jedisct1/libsodium/releases/download/1.0.19-RELEASE/libsodium-1.0.19.tar.gz
# ... configure and make ...

# Download and compile libzmq
RUN wget https://github.com/zeromq/libzmq/releases/download/v4.3.4/zeromq-4.3.4.tar.gz
# ... configure and make ...

# Copy project and compile via make
COPY . .
RUN make  # This runs: go build ... cmd/main.go

# Stage 2: Runtime (minimal image)
FROM gcr.io/distroless/cc-debian11:nonroot
COPY --from=build /go/bin/fleet-telemetry /
CMD ["/fleet-telemetry", "-config", "/etc/fleet-telemetry/config.json"]
```

## 🚨 Troubleshooting

### Build Fails
```bash
# See full build output
docker-compose -f docker-compose.postgres.yml up --build

# Clear build cache and rebuild
docker system prune -a
docker-compose -f docker-compose.postgres.yml up -d --build
```

### Database Connection Issues
```bash
# Check PostgreSQL is running
docker-compose -f docker-compose.postgres.yml logs postgres

# Test connection manually
psql -h localhost -U telemetry_user -d fleet_telemetry
```

### Port Already in Use
```bash
# Find what's using the port
lsof -i :5432  # PostgreSQL
lsof -i :443   # Fleet Telemetry
lsof -i :8080  # Status endpoint
lsof -i :8000  # Metrics

# Kill the process or change port in docker-compose.postgres.yml
```

### TLS Certificate Issues
```bash
# Regenerate certificates
go run tools/main.go

# Verify they were created
ls -la test/integration/test-certs/
```

## 🔐 Production Notes

### Current Setup (Development)
- Uses self-signed TLS certificates
- PostgreSQL password in plain text (docker-compose file)
- Data expires when container is removed

### For Production
1. **Use proper TLS certificates**
   - From Let's Encrypt or your CA
   - Update `test/integration/test-certs/`

2. **Secure secrets**
   - Use Docker secrets or environment files
   - Don't commit passwords to git

3. **Persistent storage**
   - Already using PostgreSQL volume ✓
   - Add backup strategy

4. **Monitoring**
   - Prometheus metrics enabled ✓
   - Add alerting on top of metrics
