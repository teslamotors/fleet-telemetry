Docker Compose Build Chain:
┌──────────────────────────────────────────────────────────────┐
│ docker-compose up -d                                         │

│ Reads docker-compose.postgres.yml                           │

│   context: .        → Use fleet-telemetry directory         │

│     ↓                                                        │
│ Executes Dockerfile:                                        │
│   Stage 1: Build Container                                 │
│     ├─ golang:1.23-bullseye base image                    │
│     ├─ Downloads libsodium-1.0.19 → compile              │
│     ├─ Downloads zeromq-4.3.4 → compile                  │
│     ├─ COPY . . (copies project)                         │
│     ├─ RUN make (builds via Makefile)                    │
│     │   └─ go build -o /go/bin/fleet-telemetry cmd/main.go
│     └─ Result: /go/bin/fleet-telemetry binary            │
│                                                            │
│   Stage 2: Runtime Container                              │
│     ├─ gcr.io/distroless/cc-debian11:nonroot base        │
│     ├─ COPY binary from Stage 1                          │
│     └─ CMD ["/fleet-telemetry", "-config", ...]         │
│     ↓                                                        │
│ Creates: fleet-telemetry:latest (minimal ~50MB image)     │
│     ↓                                                        │
│ Starts Containers:                                         │
│   1. fleet-telemetry-postgres (PostgreSQL 15)            │
│   2. fleet-telemetry-app (Fleet Telemetry)               │
│                                                            │
│ Configures Networking:                                    │
│   ├─ Both on fleet-telemetry-network                     │
│   ├─ App → DB communication via service name             │

🚀 QUICK START VERIFICATION
════════════════════════════════════════════════════════════════════

Pre-flight Checklist:

  ☐ 1. Certificates generated
       Command: go run tools/main.go
       Verify: ls -la test/integration/test-certs/

  ☐ 2. PostgreSQL config exists
       File: test/integration/postgres_config.json
       Verify: cat test/integration/postgres_config.json | grep postgres

  ☐ 3. Docker installed
       Command: docker --version

  ☐ 4. Docker Compose installed
       Command: docker-compose --version

Start Sequence:

  ☐ 1. Navigate to project
       cd /home/user/share/fleet-telemetry

  ☐ 2. Build and start
       docker-compose -f docker-compose.postgres.yml up -d

       First run: Takes 5-10 minutes (builds dependencies)
       Next runs: Takes < 1 minute (uses cache)

  ☐ 3. Check status
       Command: docker-compose -f docker-compose.postgres.yml ps
       Expected:
         • fleet-telemetry-postgres: Up (healthy)
         • fleet-telemetry-app: Up

  ☐ 4. Verify app is running
       Command: curl http://localhost:8080/status
       Expected: JSON response with status info

  ☐ 5. Check database
       Command: docker exec fleet-telemetry-postgres \
                psql -U telemetry_user -d fleet_telemetry \
                -c "SELECT COUNT(*) FROM tesla_V;"
       Expected: rows with data count (starts at 0)

🔄 DEVELOPMENT WORKFLOW
════════════════════════════════════════════════════════════════════

Scenario 1: Code Change
────────────────────────
  1. Edit your code
     vim datastore/postgres/postgres.go

  2. Rebuild and restart
     docker-compose -f docker-compose.postgres.yml up -d --build

  3. See logs
     docker-compose -f docker-compose.postgres.yml logs -f app

Scenario 2: Config Change
────────────────────────
  1. Edit config
     vim test/integration/postgres_config.json

  2. Restart app
     docker-compose -f docker-compose.postgres.yml restart app

  3. See logs
     docker-compose -f docker-compose.postgres.yml logs -f app

Scenario 3: Database Issue
────────────────────────
  1. Check PostgreSQL logs
     docker-compose -f docker-compose.postgres.yml logs postgres

  2. Connect to database
     docker exec -it fleet-telemetry-postgres \
     psql -U telemetry_user -d fleet_telemetry

  3. Run queries
     SELECT COUNT(*) FROM tesla_V;

Scenario 4: Full Restart
────────────────────────
  1. Stop everything
     docker-compose -f docker-compose.postgres.yml down

  2. Remove volumes (data)
     docker-compose -f docker-compose.postgres.yml down -v

  3. Start fresh
     docker-compose -f docker-compose.postgres.yml up -d

📝 DOCKERFILE EXPLANATION
════════════════════════════════════════════════════════════════════

Why Multi-Stage Build?

Stage 1 - Build (Heavy):
  • Full Go compiler
  • Build tools
  • Source code
  • Intermediate artifacts
  • Total: ~1GB+

  → Used only to compile the binary

Stage 2 - Runtime (Light):
  • Only the compiled binary
  • Minimal OS (distroless)
  • No build tools
  • No source code
  • Total: ~50MB

  → Used in production container

Benefit: Small, secure, fast to distribute
